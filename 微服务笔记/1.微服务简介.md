# 微服务简介

微服务是相对于单体式服务而言的，一般情况下为适应需求，需要把大型复杂的系统不仅仅从逻辑上，更是从物理上进行拆分成一个个独立的小型服务。

## 为什么要拆呢？

单体式应用，内部虽然在逻辑上有一定分层（如MVC架构），但在物理上是耦合的。
随着体积越来越大导致：

1. 复杂性越来越高
2. 积累技术债
3. 维护成本高
4. 交付时间长
......一系列问题

## 微服务的特性

1. 单一职责
2. 轻量通信
3. 独立性强
4. 进程隔离

## 微服务的缺点

1. 运维广度要求高
2. 分布式，增加复杂性
3. 接口调整成本比单体架构要高（虽然单体架构也不低）
4. 重复劳动，微服务各自为战，难免需要重复造轮子。单体架构到是可以省。但是我宁愿空间换时间。
